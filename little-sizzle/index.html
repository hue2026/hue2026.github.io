📖 Vocabulary

Key Terms — Lesson 1

🐱 Sprite

Any character, object, or shape in your Scratch project. Can move, animate, and react to code.

🖼️ Backdrop

The background image of the Stage. Changes scenery without affecting sprites. Like a movie set.

🧩 Block-Based Coding

Programming by snapping visual blocks together — no typing! Each block = one instruction.

🔢 Algorithm

A step-by-step set of instructions. In Scratch, a stack of blocks = an algorithm. Order matters!

⚡ Interactive

A project that responds to player input — keyboard keys, mouse clicks, timer events.

🎬 Animation

Using the Repeat block to cycle through costumes (poses) so a sprite appears to move.

🎭 Costume

Different "looks" for a sprite. Switching between costumes creates animation frames.

💭 Quick check: An algorithm is like a recipe — what happens if you follow the steps in the wrong order?

🧩 Block Categories

The Building Blocks of Your Game

🏃 Motion

Move sprites around the stage. Change X/Y position, point in direction, glide to coordinates.

move 10 stepschange x by 10

⚡ Events

Trigger code when something happens — green flag clicked, key pressed, sprite clicked.

when 🏁 clickedwhen [space] key pressed

🔁 Control

Loop code (repeat, forever), conditionals (if/else), wait, stop. Controls flow of your program.

foreverrepeat 10if <> then

🔊 Sound

Play sounds, control volume. Over 100+ built-in sounds. Essential for game feedback!

play sound [pop]

👁️ Looks

Change costume, show/hide sprite, set size, say something. Controls visual appearance.

switch costume tosay [Hello!] for 2 sec

🎯 Sensing

Detect if touching a sprite/color, mouse position, key presses. Critical for collision in games!

touching [sprite]?

🔢 Algorithms

Thinking Like a Programmer

📋 A Simple Movement Algorithm

1
when 🏁 clicked

The "trigger" — sets everything in motion

2
forever

Keep checking — the game loop runs 30x/sec

3
if <key [→] pressed?> then

Check user input before acting

4
change x by 10

Execute the action — sprite moves right

💭 What happens if you remove the forever block? Would the game still work?

🍳 Algorithm = Recipe

Making a sandwich:

1

Get bread

2

Spread peanut butter

3

Put bread on top

What if you do Step 3 first? Algorithms require correct order!

⚡ Key Insight

In Scratch, blocks are read top-to-bottom. The computer executes EXACTLY what you tell it — nothing more, nothing less. Bugs = wrong instructions!

🎮 Game Design

What Makes a Game a Game?

🎯 Goal / Objective

Every game has a purpose. Reach the finish line, collect all stars, survive waves of enemies. Without a goal, it's a toy — not a game.

🕹️ Player Input

How the player controls the game. Keyboard, mouse, touch. Your Event + Motion blocks create this!

🔄 Game Loop

The heart of any game. Check input → Update state → Display result → Repeat. The forever block IS the game loop in Scratch!

💥 Feedback

Sound effects, score changes, visual reactions. Tells the player: "Yes! That worked!" Critical for engagement.

⚖️ Challenge Curve

Too easy = boring. Too hard = frustrating. Great games start easy and get progressively harder.

🌍 Game World

Your backdrop, sprite designs, and sounds. Creates the atmosphere and pulls players into your universe.

🔍 Why Each Block Matters

when 🏁 clicked

The starter. Without it, code never runs. It's the game's power button.

forever

The game loop. Keeps checking input ~30 times/second. Stops only when you click Stop.

if <> then

The decision maker. Only runs the code inside IF the condition is true.

change x by

The action. Positive = move right. Negative = move left. X is the horizontal axis.

Moving Sprites with Arrow Keys

Scratch — Programming Arrow Keys to Move Sprites

Two Methods Shown:

Method 1: Event Blocks

Separate "when key pressed" for each direction. Simple but slight delay between presses.

Method 2: Forever + Sensing

Uses forever + if <key pressed?> — smoother movement, perfect for games!
